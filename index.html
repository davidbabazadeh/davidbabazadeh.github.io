My First Project

This is the first project I made in this class, I've coded one other project before, but it was only text-based. This class was my first experirience using visual programmming and I had a lot of fun with it. In this project I learned about what the "canvas" is, what its properties are, and how to play with it in javascript. I also learned about using keyboard input and using animation loops. I've only done object oriented programming once before so this project was a step up, same with algorithms and loops. And finally, I figured out how collision detection works with rectangular figures.

"R" to start; "wasd" to move; "shift" to sprint; 1234 to change difficulty between games

later on I made a upgraded version...
in this version everything is controlled by the mouse and you can't sprint, I was able to put use my knowledge on trigonometry to make sure there is a max speed for the player to move at all times in any direction.

Though these two are the least complicated projects i've worked on, I think they're the most fun, some people I've showed it to actually found it addictive.

Atari Adventure Remake

This project started when I wanted to try using a 2d array (or matrix) to make a map that moves around the play which stands still. When my cs teacher, Mr. Landgreen, saw it, he said it reminded him of atari adventure from when he was a kid and challenged me to try and create it and offered extra credit, it was a challenge for me at the time, in the beginning of the year, but I didn't turn down the challenge and succeeded.
In this project I learned a lot more about creating algorithms, iterating with a loop to move through both dimensions of the map, creating collision for each block, etc. The biggest challenge was making the program flexible for different fields of view (FOV) which you can change w=using the console.

wasd to move; e to pick up, space to drop; if the dragon catched you, you'll need to refresh the page!

"Sky Gunner"

I made this project because I wanted to make a simple game where It would be easy to change what pictures are used in html so you can change what's going on.
In this project I experimented with accelerations for moving left and right and jumping, though jumping doesn't have a real purpose it was fun playing with it, I also experimented with looping though an array to check seperate arrays for collisions in order to check when bullets or enemies shoud dissappear.

wasd to move; s to start/restart; l to shoot
