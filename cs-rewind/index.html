Atari Adventure Remake

This project started when I wanted to try using a 2d array (or matrix) to make a map that moves around the play which stands still. When my cs teacher, Mr. Landgreen, saw it, he said it reminded him of atari adventure from when he was a kid and challenged me to try and create it and offered extra credit, it was a challenge for me at the time, in the beginning of the year, but I didn't turn down the challenge and succeeded.
In this project I learned a lot more about creating algorithms, iterating with a loop to move through both dimensions of the map, creating collision for each block, etc. The biggest challenge was making the program flexible for different fields of view (FOV) which you can change w=using the console.

wasd to move; e to pick up, space to drop; if the dragon catched you, you'll need to refresh the page!

~1 month: { 15% research; 35% writing code; 45% debugging; 5% styling }

When playing with arduinos I tried attaching a bunch of different devices, LEDs, potentiometer, LED strips, joysticks, pressure sensors, servo motors, stepper motors, remotes, IR sensors, and more. I learned a lot about how to read and use schematics, even applying only similar ones to your situation, which taught me about circitry.

First Arduino Project

In our first project all we did was attach an led to a potentiometer and the arduino so we could use the knob to change the brightness of our LED. It wasn't much, but it was a first step into learning about the hardware we were creating.

~2 days: { 65% research; 5% writing code; 5% debugging; 25% wiring }

Joysticks!

Experimenting with the joysticks was a lot of fun because it was an interesting source of input. Once we got it working through the software we were easily able to attach it to a servo to control the direction it was pointing. Our first cool project with the joystick was attaching it to an LED strip and controling the position of a line and the color of it.
